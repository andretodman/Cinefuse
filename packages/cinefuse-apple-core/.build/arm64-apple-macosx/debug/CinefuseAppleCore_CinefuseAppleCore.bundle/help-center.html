1) Overview

Cinefuse is an MCP-first film editor. The app sends requests to the API gateway, which dispatches generation through MCP tools and the render-worker queue.

2) Quick Start Workflow

Video Creation

  1. Create a project from the top bar.
  2. Add one or more shots with detailed prompts.
  3. Quote then generate to queue a render job.
  4. Watch progress and status in shot cards and the jobs grid.
  5. Add audio lanes and run stitch/export when ready.

Audio Creation (Creation → Audio)

  1. Optional: build Sound blueprints on the left (reference audio/video → Save blueprint).
  2. Draft in Sounds: prompt → tier → Sound blueprints toolbar → tags → Quote → Create Sound.
  3. Select timeline clips to edit blueprint associations per sound, or leave unselected to set defaults for the next sound you create.
  4. Quote then Generate sound on each card when ready.

3) Workspace + Menu Controls

The top menu includes Creation (Video vs Audio), workspace preset, panel toggles, theme, settings, and Help.

  • Video Creation — storyboard, characters, shots, video preview, layered audio lanes (dialogue/score/SFX/mix/lipsync), and full stitch/export pipeline.
  • Audio Creation — Sound blueprints (left), Sounds panel with tier row and embedded blueprint toolbar (see §8), waveform-style timeline, audio-centric preview, and Export layered mix for a combined WAV job. Not the same UI as video-mode audio lanes.
  • Toggle side/bottom panes for focused editing.
  • Toggle Audio Lanes / Jobs independently in render (video) mode.
  • Use Help button to open this embedded docs center.

4) Projects

  • Create, rename, and select projects from the sidebar.
  • Each project stores shots, tracks, jobs, and generated outputs.
  • Project details sync via API fetch + event stream updates.

5) Shots + Prompting

Shot cards are the source of truth for visual intent and render options.

  • Prompt text is selectable/copyable.
  • Tier chooses routing behavior and expected cost band.
  • Generate queues a render job; retry is for failed shots.
  • Diagnostics reveal request stage, backend status, and provider details.
Image slot: Shot card anatomy (prompt, status, actions, diagnostics menu).

6) Timeline and Visibility

  • Timeline shows shot order and render outputs.
  • Hidden clips can be restored using the unhide controls.
  • Rendered clips can be previewed directly from cards.

7) Jobs Grid and Status

  • Jobs panel uses a grid to display multiple active items.
  • Supports retry/delete/copy diagnostics actions.
  • Status can move through queued → generating/running → done, or failed/timedOut.

Green: done/ready  • Yellow: queued/running  • Red: failed/timedOut

8) Audio Creation: Sounds + Sound blueprints

Open Creation → Audio for sound-focused work. This mode is separate from Video Creation layered audio lanes (dialogue/score/SFX/mix/lipsync on the video timeline).

Layout

  • Sound blueprints (left): project-scoped blueprints with optional reference audio/video via the file importer. Save blueprint stores them on the server as referenceFileIds.
  • Sounds panel row order: prompt → Budget / Standard / Premium (tier) → Sound blueprints toolbar (blueprint menu with checkmarks + Import reference files) → tags → Quote / Create Sound.

Timeline selection vs draft defaults

  • When a timeline clip matches the current selection, the blueprint toolbar edits the highlighted sound’s associated blueprint set.
  • When nothing on the timeline matches (or no matching clip), the toolbar edits draft defaults for the next Create Sound.

Import and blueprints

  • Import reference files uploads files, creates a blueprint, and adds it to the current selection (draft or selected timeline shot, per app behavior).
  • Use the blueprint menu for multi-select (checkmarks) among saved blueprints.

Quote and Generate

  • Quote then Generate on a sound card applies as before. On generate, optional client fields supply blueprint IDs; the gateway merges them into the shot’s audioRefs when the job runs.

9) Audio Lanes (video timeline)

In Video Creation, these are layered lanes on the video timeline—not the same surface as Audio Creation → Sounds.

  • Dialogue, score, and SFX generation lanes.
  • Mix and lipsync passes can be queued as jobs.
  • Audio outputs are attached to timeline context.
  • Successful generations persist remote URLs; the app syncs audio artifacts into Documents → Cinefuse Generated → <project> alongside video exports.
  • If an upstream provider cannot fulfill a specific audio tool, the job may record a skipped feature (zero Sparks debited). Open the status/diagnostics panel on that job for provider, reason, and whether any file was still produced.

10) Stitch + Export

  • Stitch creates a coherent sequence from shot outputs.
  • Export supports resolution, captions, and publish/archive options.
  • Export jobs appear in the same jobs track for visibility.

11) Diagnostics + Failure Recovery

  • Queued too long: likely worker backlog/offline.
  • Retry conflict: backend says current status is no longer failed.
  • Timed out: no meaningful progress transition in timeout window.
  • Audio feature skipped: provider adapter could not complete that tool; check provider name, precise reason, and whether outputCreated is still true for partial artifacts.

MCP services involved

  • mcp/clip: quote_clip / generate_clip forward optional audioRefs (Pubfuse file IDs) into the configured provider/FAL payload where plumbing supports it; the provider must resolve IDs.
  • mcp/audio: dialogue/score/sfx/mix/lipsync generation.
  • mcp/stitch: stitch timeline segments and transitions.
  • mcp/export: finalize export package and metadata.
  • mcp/pubfuse: file/project integration with Pubfuse REST.
  • mcp/billing: Sparks quote/debit/credit flow.
